# GAPAK UX Audit & Premium Redesign

## 1. Полный UX аудит

GAPAK должен восприниматься как спокойная, статусная социальная сеть: пользователь за 3 секунды видит главный фокус, понимает действие и не отвлекается на служебную информацию. Текущий интерфейс решает продуктовые задачи, но перегружает экран объяснениями, бейджами, второстепенными метриками и разрозненными карточками.

## 2. Список проблем

- Слишком много конкурирующих карточек в первом экране: герой, метрики приватности, сигналы и блоки видимости спорят с лентой.
- Текст объясняет продукт вместо того, чтобы дать простой сценарий: открыть ленту, создать пост, смотреть клипсы.
- Технические детали публикаций (`Audience IDs`, статусы приватности, служебные бейджи) создают шум.
- Правая колонка занята настройками, а не социальным discovery.
- Клипсы используют обычную карточку поста, поэтому не ощущаются как отдельный video-first опыт.
- Палитра содержит слишком много теплых/ярких акцентов; premium-впечатление требует более холодной и контролируемой системы.

## 3. Новая дизайн-система

- **Принцип:** content-first, calm luxury, one primary action per screen.
- **Карточки:** единый glass-surface, радиус 28–36 px, тонкая белая обводка, мягкая тень, без лишних рамок внутри.
- **Типографика:** максимум 3 уровня: hero 48–60 px, section 28–36 px, body/action 13–16 px.
- **Иконки:** только для действий и навигации; не использовать иконки для объяснения очевидных блоков.
- **Микрокопи:** короткие заголовки и кнопки; все длинные маркетинговые тексты удаляются.
- **Компоненты:** Hero, Feed Post, Recommendation Row, Clip Card, Empty State, Primary CTA.

## 4. Новая цветовая палитра

- Background: `#05070D`, `#070B12`, `#0B1018`.
- Surface: `rgba(255,255,255,0.04–0.08)` с blur и тонкой обводкой.
- Text primary: `#FFFFFF` / `#E7F4FF`.
- Text secondary: `#AAB9CE`.
- Accent primary: cold cyan `#67E8F9`.
- Accent secondary: soft violet `#8A7DFF`.
- Accent blue: `#67D0FF`.
- Запрещённые акценты: acid yellow, acid pink, saturated neon glow.

## 5. Новая структура главной страницы

1. Большой hero на всю ширину контентной области.
2. Один заголовок: «Контролируй свою цифровую жизнь».
3. Один короткий подзаголовок.
4. Главная кнопка «Начать» и вторичная ссылка на клипсы.
5. Лента как основной фокус ниже hero.
6. Правая колонка только для рекомендаций публичных аккаунтов.

## 6. Новая структура ленты

- Автор: аватар, имя, username.
- Время публикации.
- Контент.
- Действия: лайк, комментарий, репост.
- Удалены технические данные, Audience IDs, длинные статусы, лишние бейджи, save-кнопка и повторяющиеся подписи.

## 7. Новая структура блока рекомендаций

- Заголовок «Рекомендуемые».
- Только публичные аккаунты.
- Компактная строка: аватар, имя, username, followers, кнопка «Подписаться».
- Поведение визуально ближе к Instagram / Threads / X: быстро сканируется, не перегружает экран.

## 8. Новая структура клипсов

- Отдельная страница с video-first заголовком.
- Сетка вертикальных карточек 9:16.
- Видео занимает почти всю карточку.
- Действия поверх превью: лайк, комментарий, репост.
- Текст ограничен двумя строками поверх нижнего gradient overlay.

## 9. Улучшения мобильной версии

- Hero получает компактные отступы и не перегружает первый экран.
- Правая колонка рекомендаций скрывается на малых экранах, чтобы не мешать ленте.
- Карточки постов имеют крупные tap targets и минимум текста.
- Клипсы в одну колонку с вертикальным просмотром; на планшетах — две колонки.
- Все действия доступны без горизонтального скролла.

1. Обновить токены в `globals.css`: графитовый фон, dark glass surfaces, cyan/violet accents.
2. Нормализовать `Card`, `Button`, `Badge`: единые радиусы, тени, фокус-состояния.
3. Пересобрать `/feed`: hero + content feed + recommendations.
4. Упростить `PostCard`: удалить служебные поля и оставить только core social actions.
5. Пересобрать `/clips`: отдельная video-first сетка и overlay-действия.
6. Добавить adaptive responsive rules Tailwind: `xl` для сайдбара, `md/2xl` для клипсов.
7. Провести проход по copy: убрать длинные объяснения, оставить короткие CTA.
8. Проверить lint/typecheck/build.
9. Сделать visual QA в браузере: первый взгляд, hierarchy, mobile, empty/error states.
10. Постепенно заменить статические рекомендации реальным API публичных аккаунтов.
